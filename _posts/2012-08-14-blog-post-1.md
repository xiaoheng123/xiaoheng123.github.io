# 1.ATRACE框架层打点

android内置atrace接口允许在框架层代码中手动添加自定义的跟踪点，以捕获特定代码段的执行情况。以下是如何在 Android 代码中增加 Atrace 跟踪的方法：

### 1.Java Framework层：

`import android.os.Trace;`

`Trace.traceBegin(long TraceTAG, string msg);`

`Trace.traceEnd(long TraceTAG);`

### 2.Native Framework层

`#define ATRACE_TAG ATRACE_TAG_GRAPHICS`设置atrace类型标签

`#include<utils/Trace.h>`

1.`ATRACE_CALL();`

等价于`ATRACE_NAME(function_name)`,trace范围为调用处到函数结束

2.`ATRACE_BEGIN(const char* msg);`

`  ATRACE_END();`

3.`ATRACE_NAME(const char* msg)`，trace范围为调用处到函数结束

如需要的trace信息中包含一些参数（如`uint64_t transactionId`）

```c++
std::string str= "setTransactionState"+std::to_string(transactionId);
const char* cstr = str.c_str();
ATRACE_NAME(cstr);
```

### 2.adb shell 命令抓取

```plain&#x20;text
adb shell atrace view am wm freq idle sync sched aidl gfx -a 要监控的app名 -t 6 >  保存的文件名.trace
```

抓取后可用adb pull命令把trace文件拉到本地

`adb pull /data/local/traces/ .`

在perfetto首页点击open trace file可以可视化trace
